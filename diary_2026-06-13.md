## 🏫 오후 1: 급식 API 연결

**급식 API 키**를 가지고 있다고 해서 연결 작업을 시작했어요.

처음엔 "API 키 자리만 만들어달라"고 해서 `MEAL_API_CONFIG` 설정 블록을 만들었어요.
그런데 키를 직접 `app.js`에 넣으면 **브라우저에서 키가 보여서 위험**해요.
그래서 바로 보안 문제를 해결하기로 했어요.

**해결 방법: 서버 프록시 방식**
- API 키는 `.env` 파일(서버 전용)에만 저장
- 브라우저는 `/api/meal`, `/api/timetable`만 호출
- 서버가 대신 NEIS에 요청하고 결과만 돌려줌
- `.gitignore`에 `.env` 추가해서 실수로 올라가는 것도 방지

`server.js` 파일을 새로 만들고, `npm start`로 바로 실행되게 했어요.

## 🔐 오후 2: 보안 점검

**OWASP 10대 보안 취약점** 기준으로 앱 전체를 점검했어요.

| # | 취약점 이름 | 쉽게 말하면 | 우리 앱 상태 |
|---|---|---|---|
| 1 | 접근 제어 취약점 | 허락 없이 들어가는 문제 | ✅ 해당 없음 (로그인 없는 앱) |
| 2 | 암호화 실패 | 비밀을 안 숨기는 문제 | ✅ API 키 `.env`로 분리 완료 |
| 3 | 인젝션 | 나쁜 명령어 끼워넣기 | ✅ `escapeHtml()`, `normalizeText()` 적용 |
| 4 | 안전하지 않은 설계 | 처음부터 위험하게 만들기 | ✅ 프록시 구조로 안전 설계 |
| 5 | 보안 설정 오류 | 서버 설정이 엉성한 문제 | ✅ `.env`에서만 키 읽음 |
| 6 | 취약한 컴포넌트 | 낡은 부품 쓰는 문제 | ✅ 순수 Node.js (외부 패키지 없음) |
| 7 | 인증/세션 문제 | 아이디 검증 허술 | ✅ 해당 없음 (현재 로그인 없음) |
| 8 | 소프트웨어 무결성 | 파일 변조 탐지 문제 | ✅ 입력값 길이·형식 검증 적용 |
| 9 | 로깅/모니터링 부족 | 이상한 일 놓치는 문제 | 🟡 `console.warn`만 있음 (개선 여지) |
| 10 | 서버 사이드 요청 위조 | 서버를 속여 요청 보내기 | ✅ API URL 고정 (외부 입력으로 변경 불가) |

**주요 보안 조치 요약:**
- `escapeHtml()` → 화면에 출력할 때 나쁜 코드가 실행되지 않게 막아줌 (XSS 방지)
- `normalizeText()` → 제어문자 제거, 길이 제한
- 입력값 빈 값/형식/범위 검증
- localStorage 로드 시 형식 검증 (`sanitizeData`)
- 브라우저 코드에서 API 키 0개 확인됨

## 🏫 오후 3: 샘플 데이터 삭제 + 학교 선택 기능

**"가상 캐릭터 샘플을 지워달라"** 고 해서 모든 가짜 데이터를 삭제했어요.

지운 것들:
- 가짜 시간표 (국어·수학·과학 수업)
- 가짜 급식 (흑미밥·미역국·닭갈비 등)
- 가짜 공지 (현장체험학습 안내 등)
- 가짜 프로필 (백준기, 대구왕선초등학교)

그리고 예전에 저장된 데이터도 자동으로 지워지게 만들었어요.
앱을 다시 켜면 스스로 확인하고, 샘플이 남아있으면 자동 삭제됩니다.

---

마지막으로 **"내 정보에서 학교를 선택하면 그 학교 자료가 보이게"** 해달라고 해서 구현했어요.

- 내 정보 탭에 **학교 선택 드롭다운** 추가
- 선택하면 시간표·급식 캐시 초기화 후 그 학교 기준으로 재조회
- 서버에 `/api/schools` 엔드포인트 추가
- `.env`에 `SCHOOL_OPTIONS_JSON` 설정으로 여러 학교 등록 가능
